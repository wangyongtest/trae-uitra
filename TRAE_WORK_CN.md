---

## 一、产品定位

TRAE WORK CN 是字节跳动推出的 **AI 原生工作台**，提供网页版、桌面版、移动版三端联动，设有 Work（办公）与 Code（编码）双模式，覆盖从专业开发到日常办公的全场景 AI 协作体验。

- **品牌归属**：TRAE 产品家族（TRAE IDE / TRAE Work / TRAE CLI / TRAE Plugin）
- **前身**：TRAE IDE SOLO 模式（独立化后成为 TRAE Work）
- **核心能力**：云端智能体、多模型支持、MCP 生态、Skills 技能系统、跨设备任务调度
- **目标用户**：开发者、产品经理、数据分析师、运营人员等全角色

## 四、核心能力架构

### 4.1 Agent 系统

#### SOLO / Code 模式 Agent
```
┌─────────────────────────────────────────────────────────────┐
│                    SOLO Agent Loop                          │
├─────────────────────────────────────────────────────────────┤
│  用户输入 → 需求理解 → 上下文构建 → 任务规划                │
│       ↑                                         │          │
│       │                                         ▼          │
│  结果验收 ← 产物预览 ← 执行测试 ← 代码生成 ← 工具调用       │
│       │                                         │          │
│       └────── 反馈修正 ←────────────────────────┘          │
└─────────────────────────────────────────────────────────────┘
```

**关键特性：**
- **Spec & Plan 工作流**：先规划再执行，计划需用户审批
- **多任务并行**：支持同时运行多个 Agent 任务
- **Auto 模式**：AI 自主推进，自动审批工具调用
- **YOLO 模式**：跳过所有审批，全速执行
- **Ask 模式**：每次工具调用需用户确认

#### 云端智能体（Cloud Agent）
- **统一运行时**：所有代码在稳定隔离的远程环境中执行
- **环境一致性**：避免本地环境差异导致的兼容性问题
- **弹性资源**：按需分配计算资源
- **跨设备接续**：本地设备离线自动切换到云端执行

## 七、关键设计决策总结

| 决策 | TRAE Work 选择 | 理由 |
|------|---------------|------|
| 产品形态 | 三端（Web/Desktop/Mobile） | 覆盖全场景、跨设备调度 |

## 八、优势与局限

### 优势
1. **全场景覆盖**：Work + Code 双模式，开发+办公通吃
2. **三端联动**：Web/Desktop/Mobile 无缝同步，移动调度中心
3. **云端能力**：Cloud Agent 弹性环境，跨设备接续
4. **完整 IDE 能力**：编辑器/调试/Git/终端/插件，生产力工具链完整
5. **企业级能力**：SSO、审计、用量统计、MCP 白名单等
6. **MCP 生态**：全面支持 MCP 协议，热门 MCP 开箱即用
7. **CUE 补全**：深度代码补全，支持主流语言
8. **多模型支持**：内置多种模型 + 自定义 API Key
9. **飞书/GitHub/Figma 集成**：主流工具链打通
10. **语音交互**：移动端语音输入，桌面端语音讨论

### 局限
1. **缓存优化非首要目标**：相比 Reasonix 99.82% 命中率，TRAE 上下文管理更注重通用性而非极致缓存命中
2. **成本透明度不足**：缺乏每轮实时费用显示、缓存命中率监控面板（Reasonix 有 StatsPanel）
3. **闭源核心**：核心 Agent 逻辑不开放，无法二次开发内核
4. **tool-call 修复弱**：通用多模型架构，缺乏针对特定模型（如 DeepSeek R1）的后处理 pipeline
5. **模型自动路由粗放**：没有类似 `<<<NEEDS_PRO>>>` 的模型自评估动态升级机制
6. **分发依赖平台**：无单二进制 CLI 轻量分发（Reasonix ~7MB 单文件）

| 维度 | TRAE Work CN | Reasonix |
